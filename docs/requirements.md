# 要件定義

## 一言サービスコンセプト（サービスのキャッチコピーを一言で）

「スタバの売り切れ、もうハズさない」

## 誰のどんな課題を解決するのか？

-   **ターゲット**
    -   スタバの新作を楽しみにしている人
    -   仕事・学校帰りにスタバへ寄る人
    -   売り切れ前にどうしても飲みたい人
-   **ユーザーの課題**
    -   店に行ったら売り切れていてガッカリする
    -   店舗ごとの在庫状況が分からないから把握したい
    -   正確な売り切れ情報を知りたい

## なぜそれを解決したいのか？

-   「欲しいフードとドリンクを、欲しい時に、無駄なく確実に手に入れたい」

## どうやって解決するのか？

-   **Google Maps API** を使って全国のスタバ店舗を地図上に表示
-   ユーザーが投稿する「売り切れ状況」をリアルタイムで共有
    -   まだある / もうない の 2 ステータス
-   **投稿には有効期限（例：1 週間）** を設定し、古い情報は非表示 or 薄表示
-   店舗アイコンの色を変えて一目で状態が分かるようにする（ドリンクかフードを前に選択させる）
-   投稿は口コミ形式で誰でも簡単にでき、情報が集まりやすい

## 機能要件

### 1. 認証機能

-   ユーザー登録、ログイン、ログアウト

### 2. 店舗一覧（Google Maps API）

-   Google Maps 上に店舗をピン表示
-   現在地付近の店舗を表示
-   最新投稿に応じてアイコンの色を変える  
    （緑：在庫あり　/ 赤：売り切れ）

### 3. 店舗詳細ページ

-   店舗情報（名前・住所）
-   最新の投稿一覧（時間順）

### 4. 売り切れ投稿機能

-   店舗選択（地図 or リスト）
-   商品名、ステータス（在庫あり / 売り切れ）
-   コメント（任意）

### 5. 投稿の有効期限

-   1 週間以上古い投稿は古い扱い  
    （薄く表示 or 非表示）

### 信頼度機能

-   投稿に対していいねを押せる
-   いいね数に応じて信頼度がアップ

### ★ 運営・安全系

-   通報機能（誤情報・荒らし対策）
-   管理者が投稿を削除/管理できる画面

## 非機能要件

-   個人情報は閲覧できない
-   インターフェイスはデザイン崩れしない
-   画面は１秒以内に表示できるようにする
